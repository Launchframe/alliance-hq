## Summary

**My VR & web reporting**

- **My VR tracker** — New member-facing page shows your season VR progress, pending reports, and lets you submit web VR from HQ (parity with Discord `/vr` for linked commanders).
- **Season VR events** — Web submissions persist in `member_season_vr_events` with audit trail for officers.

**Commander claims & member-link help**

- **Claim invites (phases 2–5)** — Officers send targeted claim invites; owners/officers can unlink commanders; UID confirmation step and bulk claim tools reduce roster onboarding friction.
- **Officer help queue** — Search and review member-link help requests; break-glass unlink for stuck claims; deduplicated `claim_conflict` rows so officers see one actionable ticket per issue.
- **Copy & guidance** — Clearer commander name text in help flows and ask-officer prompts when roster match fails.

**Video pipeline & review**

- **Cross-device queue** — Your active upload/review job follows you from desktop to phone (and back) through review and submit — no lost context when switching devices mid-flow.
- **In-house OCR only** — Alliance setting to run roster/score OCR on HQ’s in-house engine only (skip Ashed tandem) for alliances tuning native OCR.
- **Follow-me polish** — Smoother center-tracking seek with timestamp interpolation; fixed a React anchor-ref loop that could spin the review page.

**Alliance context, device link & Discord**

- **Ashed survives alliance switch** — Personal Ashed connection is preserved when switching between alliances (no orphan cleanup wiping your credential).
- **Link a Device fix** — Pairing a mobile session now copies Ashed identity binding (`ashedUserId`) along with the encrypted token, so the phone keeps your connection after the next page load instead of showing a stale Connect Ashed banner. Users who paired before this fix should **re-link once**.
- **Maintainer alliance switching** — Platform maintainers can change active alliance context without session loss; native-alliance video queue routing fixed on context change.
- **Discord UID-only `/link-commander`** — Commander link on Discord now matches web: enter player UID, confirm the resolved identity, then link — no typed in-game name. Hosted guide and pt-BR copy updated.

## Breaking changes

- **Discord `/link-commander` options** — The `name` option is removed; binding uses **UID + identity confirm** only (same as HQ web onboarding). Re-register bot commands after deploy (`npm run discord:register-commands`).
