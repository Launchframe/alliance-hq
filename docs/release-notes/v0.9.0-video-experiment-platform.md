## Working notes

- **PR #27:** DB-managed parse configs (`video_parse_configs`); experiment campaigns with arms, traffic percent, score-target scope.
- **Assignment:** Upload groups routed to experiment arms; control arms skip shadow passes; deterministic assignment per group.
- **Admin — Parse configs:** CRUD with layman guidance for mode, sceneThreshold, sampleFps, passKey.
- **Admin — Experiments:** Campaign list/detail, arm picker from configs, min-sample warnings, conclude → promote winner via config assignments API.
- **Admin — Video job analytics:** Aggregate pass comparison, quality buckets, recommendation vs user selection (fleet-level view from #26 feedback gap).
- **Real Steel:** Control arms cannot carry variant parse configs; activation validates arm shape.

## Summary

- **Parse configs in the database** — platform admins manage extraction settings (mode, scene threshold, sample FPS, pass key) with plain-language guidance instead of env-only defaults.
- **A/B experiment campaigns** — create campaigns scoped to a score target and board; assign traffic percentage; add arms from named configs with a control baseline.
- **Promote winners to production** — conclude a campaign and promote an arm via config assignments so new uploads use the winning parameters (existing jobs are not re-processed).
- **Fleet analytics** — `/admin/video-jobs/analytics` aggregates pass comparison, quality buckets, and user selection vs system recommendation across jobs.
- **Cost-aware experimentation** — control arms skip shadow passes; traffic gating limits experiment load on the pipeline.

## Platform maintainer notes

- Deploy runs `db:prepare`, which applies migration `0022_experiments` (parse configs, experiment campaigns/arms, upload group experiment columns).
- New platform-admin surfaces: `/admin/parse-configs`, `/admin/experiments`, `/admin/video-jobs/analytics`; APIs under `/api/admin/parse-configs`, `/api/admin/experiments`, `/api/admin/config-assignments`, `/api/admin/video-jobs/analytics`.
- Promoting an experiment winner creates a **config assignment** for the campaign’s score target — does not re-process existing uploads.
- Run separate campaigns per **score target** (event format); mixed-format samples are flagged in the UI.
