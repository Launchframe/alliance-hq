## Working notes

### Deposit slip OCR
- **PR #353 — Auto-merge duplicates:** Auto-merge exact duplicates, locked→matured/looted lifecycle pairs, and missing-timestamp twins into a non-blocking section.
- **PR #354 — Line-bbox association:** Associate Deposit, outcome, and timestamp fields by line-bbox vertical proximity instead of OCR reading-order look-ahead.
- **PR #355 — Y-band + post-loot peel:** Midpoint y-bands with full-geometry gate; peel ≤15m post-terminal re-deposits from proximity merge.
- **PR #356 — Officer member select:** Searchable member picker when OCR does not match; persist officer picks through submit/commit.

### VR, video & device link
- **PR #362 — VR anomaly threshold:** Drop legacy 10,250 OFFICER_REVIEW_THRESHOLD; anomaly confirm uses peer-gap rule only (Season 5 institute ladders exceed 10k).
- **PR #342 — Stale submitting jobs:** Recover stale `submitting` video jobs back to review so officers can retry.
- **PR #352 — Device link:** Allow re-link on the same phone session; stop leaking raw SQL errors on the pairing page.

### Admin & i18n
- **PR #351 — Admin reprocess dialog:** Reprocess dialog with FPS ladder and ad-hoc experiments.
- **PR #363 — i18n:** Correct reprocess FPS scroll-speed hints (en-US + pt-BR).
- **Discord PT:** Shorten `set-banking-channel` command translation.

## Summary

- **Deposit slip OCR** — Better field-to-row association, auto-merge of obvious duplicates and lifecycle pairs, and an officer member picker when OCR cannot match a commander.
- **VR threshold** — Stop flagging legitimate Season 5 VR bumps above 10,250; anomaly review now uses the peer-gap rule only.
- **Video submit recovery** — Stale submitting jobs return to review so you can retry instead of staying stuck.
- **Device linking** — Re-scan a new QR from an already-linked phone session; pairing errors no longer expose raw database messages.
- **Admin reprocess** — FPS ladder and ad-hoc experiment controls in the video reprocess dialog.
