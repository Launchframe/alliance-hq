## Summary

**Commanders & roster**

- **Commanders index** ‚Äî New `/commanders` page for officers: sortable THP roster, main-squad filters, squad summary strip, and a **takedown team builder** (same snake-draft logic as Discord `/takedown-teams`) for balanced rally teams.
- **Main squad** ‚Äî Linked commanders self-report aircraft/tank/missile on their profile; officers can override. Source is tracked (`self_report` vs `officer_override`).
- **Commander identity** ‚Äî UID-keyed commander records with dual-write to the alliance roster, commander-first reads, and officer tools to resolve sync conflicts when UID data is missing or ambiguous.
- **Member link polish** ‚Äî Roster-miss flows, setup-guide wiring, and safer substring suggestions when a typed name uniquely matches one roster row.

**Video pipeline & review**

- **Processor settings UI** ‚Äî Owners assign up to two video processors from a mode-aware candidate pool (`/tools/video-processors`); all members can view the roster read-only.
- **Dev/e2e OCR modes** ‚Äî `VIDEO_OCR_PROVIDER=local|mock` (with `VIDEO_OCR_ALLOW_NONPROD=true`) runs roster/score OCR without Ashed for local testing; production default remains Ashed unless explicitly overridden.
- **OCR quality observability** ‚Äî Roster OCR eval snapshots feed admin analytics and experiment detail (tandem Ashed + native metrics for tuning in-house OCR).
- **Review UX** ‚Äî Sticky preview improvements, local persistence of extracted-data edits, scroll-to-seek on the preview, **Follow me** auto-seek while scrolling the review table, and clearer mobile/desktop review layout.

**Discord bot & HQ account**

- **Clearer link commands** ‚Äî `/link` is HQ account OAuth only; `/link-commander` (alias `/link-last-war-profile`) binds your in-game character; `/link-ashed` connects optional Ashed credentials (renamed from `/link-to-ashed-seat`).
- **Multi-server self-service** ‚Äî Alliance settings panel with one-click bot install URL; owners, R4+ officers, and maintainers can register additional Discord servers (`/link-alliance`).
- **Account Discord link** ‚Äî Link or unlink Discord from HQ account settings; post-/link onboarding funnels guide the next step after HQ or commander link.
- **Hosted operator guide** ‚Äî Public `/guides/discord-bot` with role-based flows (owner vs officer vs member) and troubleshooting for missing slash commands.

**Productivity & polish**

- **Keyboard hotkeys** ‚Äî `Mod+K` command palette, full shortcut reference at `/settings/hotkeys`, server-synced custom bindings, chord/sequence support, and train-dashboard shortcuts for power users.
- **Enter to submit** ‚Äî Supported dialogs submit on Enter (desktop) or Send (mobile) when focus is not in a textarea.
- **Sign-in reliability** ‚Äî Discord OAuth authorize is session-aware (avoids `OAuthAccountNotLinked` when already signed in); Playwright no longer clobbers `.env.local` on e2e runs.

## Breaking changes

- **Discord slash commands renamed/split** ‚Äî `/link` no longer performs name+UID commander binding or Ashed credential setup. Use **`/link-commander`** (or `/link-last-war-profile`) for in-game character link and **`/link-ashed`** for optional Ashed credentials. Re-register bot commands after deploy (`npm run discord:register-commands`).
- **Any Ashed alliance member may link bot credentials** ‚Äî `/link-ashed` is available to any linked Ashed alliance member (not only the owner), for alliances still on Ashed-powered roster tools.
