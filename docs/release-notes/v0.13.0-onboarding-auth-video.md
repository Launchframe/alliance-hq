## Summary

**Onboarding & member link**

- **Self-serve owner onboarding** — A native alliance owner completes name + UID member link and the alliance automatically adopts its game server, so owners no longer need a pre-set server to get started.
- **Roster approval, by email** — Members who aren't on the roster yet are routed to the owner for approval via accept/decline email links, with reminder nudges at 24h and 72h.
- **Ops inbox** — Split inboxes surface what needs attention: failed/stuck video jobs and bug reports for platform maintainers; event schedules and upload handoffs for officers.
- **Cleaner web onboarding** — A guided name + UID link wizard, exact roster matching (no more look-alike mislinks), and standard error pages for expired links, 404s, and 500s. Signing out now fully clears alliance context so join-code tenants don't stick.

**Sign-in & account security**

- **Passwords and passkeys** — Sign in with a password or a passkey alongside magic links, managed from account security settings, plus controlled Google/Discord linking with link/unlink controls.
- **Hardened email sign-in** — Email one-time codes cap failed attempts and rate-limit requests, with alerts to maintainers if traffic spikes.
- **Account safety fixes** — Connecting Ashed can no longer hijack another HQ account, and officers keep access after signing back in without needing to reconnect Ashed.

**Video pipeline**

- **Upload vs. process roles** — Officers can queue leaderboard videos for review, while up to two designated processors per alliance approve and run OCR with their own Ashed connection — keeping shared-seat usage within Ashed's terms.
- **Bigger uploads, lower cost** — Direct-to-storage uploads remove the previous size ceiling, large videos stream through processing, and reviewed clips are compressed to 720p archives.
- **Faster, clearer review** — A sticky video preview with fit-to-width zoom for portrait boards, type-to-search member matching, add-row at top or bottom, a first-frame fix so no rows are missed, and live status updates that no longer overwrite edits you're making.

**Commanders, Discord & admin**

- **Commander profiles** — Alliance-scoped commander profiles with tenure history, server-side member search, and a platform-admin commanders console.
- **Discord `/link` by name + UID** — Linking uses in-game name + UID (Ashed optional), native owners can register their guild without Ashed, and the multi-tenant VR bot adds `/vr-report`, `/takedown-teams`, and per-guild nightly standings.
- **Train conductor commands** — Officer-gated train slash commands with announcements, departing-soon reminders, and a hosted operator guide.
- **Platform game seasons** — Maintainers can edit per-season max base VR and profession-level caps; alliances now map cleanly to servers and seasons.
- **Trains schedule fixes** — Restored 7-day week strip and forward navigation for Tuesday train weeks, correct VS-push Monday handling, and select-then-paint month scheduling.

## Breaking changes

- **HQ invites now require a linked game server** — Officer and member invites return `422 alliance_server_required` until the alliance is linked to a game server. Owner name + UID onboarding links it automatically; owners can also set/correct it in season settings, and platform maintainers via the admin alliances console. Owner invites are unaffected.
- **Video uploads no longer auto-run OCR** — Queued videos stay in `pending_approval` until a designated video processor (max two per alliance; owner/maintainer bypass the cap) approves with a live Ashed connection. Approving without one returns a recoverable `409 ashed_not_connected`.

## Platform maintainer notes

- **Migrations apply on deploy** (`db:prepare`): password/passkey auth (`0043`), roster link requests (`0051`), video processor slots + approval columns (`0053`), and the `game_seasons` / `game_servers` graph. Confirm matching `drizzle/meta/_journal.json` entries before deploy (`npm run db:validate-journal`).
- **Player UIDs are sensitive** — never displayed or logged; redacted from owner-link emails, audit metadata, and admin alerts. Name + UID is first-claim only.
- **Direct R2 uploads require bucket CORS** to be configured for browser PUTs (see the R2 CORS docs). Optional `LASTWAR_PLAYER_LOOKUP_URL` override is documented in `.env.example`.
- **Season caps** — per-season max base VR (default `10000`) and max profession level are editable at `/admin/game-seasons`.
- **Sign-in rate limiting** — `POST /api/auth/send-code` enforces per-IP and global (50/min) caps and emails maintainers (deduped per UTC minute) when the global cap blocks traffic.
